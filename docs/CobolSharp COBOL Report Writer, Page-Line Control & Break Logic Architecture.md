CobolSharp COBOL Report Writer, Page/Line Control & Break Logic Architecture (CIL‑Only)
=======================================================================================

Purpose
-------
Define the authoritative architecture for:
- REPORT SECTION structure
- RD, CONTROL, TYPE, LINE, COLUMN, SOURCE
- Page/line management
- Heading/Footing generation
- Control breaks (major/minor)
- SUM, COUNT, AVERAGE, MIN, MAX
- DETAIL vs SUMMARY lines
- Report groups and hierarchy
- Declarative exception routing
- Integration with ExecutionContext.ReportEngine
- CIL‑friendly lowering
- AOT/WASM‑safe report generation

This document governs how CobolSharp implements COBOL’s Report Writer on .NET.

------------------------------------------------------------
SECTION 1 — REPORT WRITER OVERVIEW
------------------------------------------------------------

CobolSharp implements:
- Full ISO/IEC 1989:2023 Report Writer
- Automatic page/line control
- Automatic control‑break logic
- Automatic accumulation (SUM, COUNT, etc.)
- Automatic headings/footings
- Declarative exception routing

Report Writer is executed by:
ExecutionContext.ReportEngine

Each REPORT is:
- A .NET class generated by compiler
- With metadata for groups, lines, columns
- With runtime state for page/line counters

------------------------------------------------------------
SECTION 2 — REPORT SECTION STRUCTURE
------------------------------------------------------------

2.1 REPORT SECTION
------------------
REPORT SECTION.
RD SalesReport
    CONTROL IS Region, Salesperson.

2.2 Report groups
-----------------
Types:
- PAGE HEADING
- PAGE FOOTING
- CONTROL HEADING
- CONTROL FOOTING
- DETAIL
- SUMMARY

2.3 Group structure
-------------------
Each group contains:
- LINE clauses
- COLUMN clauses
- SOURCE clauses
- SUM/COUNT/etc. clauses

Compiler generates:
- A ReportGroup object
- A list of ReportLine objects
- Column metadata

------------------------------------------------------------
SECTION 3 — PAGE & LINE CONTROL
------------------------------------------------------------

3.1 Page size
-------------
Default:
- 60 lines per page
- Configurable via PAGE LIMIT

3.2 Line counter
----------------
ReportEngine maintains:
- CurrentLine
- CurrentPage

3.3 Automatic page advance
--------------------------
If writing a line would exceed page limit:
- Emit PAGE FOOTING
- Emit PAGE HEADING
- Reset line counter

3.4 LINE clause
---------------
LINE NUMBER n:
- Absolute line
LINE PLUS n:
- Relative line

------------------------------------------------------------
SECTION 4 — COLUMN & SOURCE HANDLING
------------------------------------------------------------

4.1 COLUMN clause
-----------------
COLUMN n:
- Starting column
- 1‑based index

4.2 SOURCE clause
-----------------
SOURCE field:
- Field value copied into column
- Numeric → converted to DISPLAY
- NATIONAL → converted to DISPLAY (ASCII only)

4.3 Literal text
----------------
VALUE "literal":
- Printed verbatim

4.4 Overflow handling
---------------------
If text exceeds line width:
- Truncated
- ON EXCEPTION if configured

------------------------------------------------------------
SECTION 5 — CONTROL BREAK LOGIC
------------------------------------------------------------

5.1 CONTROL clause
------------------
CONTROL IS Region, Salesperson.

Defines:
- Major break: Region
- Minor break: Salesperson

5.2 Break detection
-------------------
On each DETAIL:
- Compare current keys to previous keys
- If Region changed → major break
- If Salesperson changed → minor break

5.3 Break sequence
------------------
On major break:
- Emit minor CONTROL FOOTING
- Emit major CONTROL FOOTING
- Emit major CONTROL HEADING
- Emit minor CONTROL HEADING

On minor break:
- Emit minor CONTROL FOOTING
- Emit minor CONTROL HEADING

------------------------------------------------------------
SECTION 6 — ACCUMULATORS (SUM, COUNT, AVERAGE, MIN, MAX)
------------------------------------------------------------

6.1 SUM
-------
SUM field INTO accumulator.

6.2 COUNT
---------
COUNT field INTO accumulator.

6.3 AVERAGE
-----------
AVERAGE field INTO accumulator.

6.4 MIN/MAX
-----------
MIN field INTO accumulator  
MAX field INTO accumulator

6.5 Reset rules
---------------
Accumulators reset:
- At start of report
- At each control break (depending on level)

------------------------------------------------------------
SECTION 7 — REPORT GROUP EXECUTION MODEL
------------------------------------------------------------

7.1 DETAIL group
----------------
Executed for each record:
- Update accumulators
- Emit DETAIL lines

7.2 CONTROL HEADING
-------------------
Executed:
- At start of report
- After control break

7.3 CONTROL FOOTING
-------------------
Executed:
- Before control break
- At end of report

7.4 PAGE HEADING / FOOTING
--------------------------
Executed automatically by page logic.

------------------------------------------------------------
SECTION 8 — CIL LOWERING RULES
------------------------------------------------------------

8.1 DETAIL lowering
-------------------
Generated IL:
- Load fields
- Update accumulators
- Call ReportEngine.EmitDetail(groupId)

8.2 CONTROL break lowering
--------------------------
Compiler generates:
- Key comparison logic
- Branches to break handlers

8.3 PAGE control lowering
-------------------------
ReportEngine handles:
- Line overflow
- Page transitions

8.4 Column rendering
--------------------
ReportEngine writes:
- Spaces up to column
- Field text
- Newline

------------------------------------------------------------
SECTION 9 — REPORTENGINE IMPLEMENTATION
------------------------------------------------------------

9.1 Responsibilities
--------------------
- Page/line control
- Control break detection
- Accumulator management
- Rendering lines
- Writing to output file
- Exception routing

9.2 Output
----------
- UTF‑8 text file
- Optional printer stream (future)

9.3 Performance
---------------
- Pooled buffers
- Zero‑allocation rendering
- AOT/WASM‑safe

------------------------------------------------------------
SECTION 10 — EXCEPTION HANDLING
------------------------------------------------------------

10.1 USE AFTER EXCEPTION ON REPORT
----------------------------------
Triggered by:
- Page overflow errors
- Invalid LINE/COLUMN
- Output file errors

10.2 ON EXCEPTION in SOURCE
---------------------------
Triggered by:
- Invalid conversion
- Overflow

10.3 ExceptionState
-------------------
Populated with:
- Report group
- Line number
- Column number
- Error message

------------------------------------------------------------
SECTION 11 — DEBUGGER INTEGRATION
------------------------------------------------------------

Debugger shows:
- Current report group
- Current page/line
- Accumulator values
- Control break state
- Rendered line preview
- ExceptionState

------------------------------------------------------------
SECTION 12 — EDGE‑CASE BEHAVIOR
------------------------------------------------------------

12.1 DETAIL with no CONTROL
---------------------------
Allowed; no break logic.

12.2 PAGE HEADING with LINE 1
-----------------------------
Always printed at top of page.

12.3 CONTROL break at end of file
---------------------------------
All CONTROL FOOTING groups emitted.

12.4 Accumulator overflow
-------------------------
Triggers ON EXCEPTION.

12.5 COLUMN < 1
---------------
Illegal.

12.6 LINE beyond page limit
---------------------------
Triggers page advance.
